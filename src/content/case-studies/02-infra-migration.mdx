## Problem

Twelve production services running on bare-metal VMs with manual provisioning. Capacity planning required a 6-week lead time. A single deploy required coordination across three teams and a step-by-step runbook.

## Constraints

- 99.95% uptime SLA on core payment services
- 4-engineer team — migration had to run alongside feature work
- No budget for a parallel environment; migration had to be incremental

## Architecture decision

Strangler fig pattern: traffic shifted service by service via weighted DNS. Terraform modules for GKE cluster, Helm charts templated from existing Docker Compose configs. Canary deploys via Argo Rollouts for payment-critical services.

## Trade-off not taken

Full Istio service mesh was scoped but deferred. Would have added 3 months to the timeline and ~15% latency overhead with our traffic pattern. mTLS at the ingress layer gave 80% of the security gain in 20% of the time.
