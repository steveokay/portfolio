## Problem

A 48-minute Jenkins pipeline was the primary velocity bottleneck. A 6% flaky test rate meant engineers routinely re-triggered builds rather than investigating failures. Every merge was a risk.

## Constraints

- Jenkins ran on-prem; no manual fallback could be required during cutover
- Test suite of 4,200 tests couldn't be split without refactoring test setup
- Flaky tests had accumulated across 3 years — root causes were mixed

## Architecture decision

Migrated to GitHub Actions with a matrix build: 8 parallel runners, test suite sharded by file checksum. Docker layer caching via GitHub Packages registry cut image pull times from 4 min to 20s. Hermetic test containers with seeded Postgres images eliminated shared-state flakiness.

## Trade-off not taken

BuildKite with faster agents benchmarked ~20% faster than GHA. Chose GHA for native integration with branch protection rules and the existing GitHub-centric review workflow. The operational overhead of a separate CI vendor wasn't justified for a 4-person team.
